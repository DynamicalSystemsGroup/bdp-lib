## Extended JSON for Front End

- The front end will need extra attributes such as positions and colors
- The [bdp-schema](https://github.com/DynamicalSystemsGroup/bdp-schema) can have extension schemas that take the base layer and extend it for this front-end version
- Two requirements here:
    - Figuring out what needs to be added for the front-end schema
    - Determining the way that we have interoperability between base and front-end
        - Could be a translation layer
        - Could also add optional attributes to base schema which would mean that it could ignore the extra features but not reject it as invalid JSON
- Two possible additional issue:
    - Re-organizing the repository to have each schema as files
    - Determining if and how to use JSON-LD with the additional schema
